# Free Debate 自由辩论模式

## 模式详述

`free_debate`（自由辩论模式）旨在促成多个独立参与者之间观点碰撞与民主共识。

**核心逻辑：**
它模拟了一个圆桌会议。不同于 `adjudication` 中法官、原告、被告的森严等级，在这里所有的参与者（Participant）地位完全平等。系统首先要求每个人背靠背（互不可见）地提出自己的初始主张，以避免先发锚定效应。随后，系统进入多轮循环的辩论期。参与者互相审阅他人观点，提出反驳或合并建议。经过充分讨论大浪淘沙后，针对存活下来的核心论点，所有人进行一次最终投票，越过多数人阈值的观点才算达成“共识”。

**适用场景：**
- 多个 AI 代理（或不同模型）对同一问题产生分歧，需要交叉讨论。
- 发散性的探讨，如：“我们微服务应该用 Monorepo 还是 Polyrepo 架构？”
- 头脑风暴与多视角方案融合。

**固定宏观阶段（Phase）：**
1. `frame`: 将问题规范化，框定辩论主题。
2. `ingest`: 摄入讨论所需的背景资料和外部上下文。
3. `initial`: 每个参与者独立（背靠背）提出自己的初始立场和主张。
4. `debate`: 循环执行的核心环节。每轮参与者会收到匿名化和去重后的同行主张及上轮摘要，并据此给出评价、反驳或合并建议。
5. `final_vote`: 辩论结束后，所有人对存活的 Active Claims 投出支持或反对票。
6. `report`: 根据投票结果（结合设定的 `vote_threshold`）产出辩论摘要与共识结果。
7. `action`: 仍然受到 Action 风险门禁的约束执行后续动作。
8. `observe`: 后验观察，记录可能的矛盾与跟进事项。

## 核心名词词汇表

| 英文单词 | 中文翻译 | 场景和用途 |
| :--- | :--- | :--- |
| **Participant** | 参与者 | 模式中的主力角色。所有人地位平等，既要抛出自己的观点，也要评价他人的观点。 |
| **Initial Claim** | 初始主张 | 在 `initial` 阶段，各个参与者背靠背（Blind）独立提出的原始观点，避免被他人带偏。 |
| **Debate Round** | 辩论轮次 | 循环执行的交锋阶段。参与者会收到匿名化、去重后的“同行主张（Peer claims）”以及上一轮摘要，借此调整自己的立场。 |
| **Judgement** | 评判 / 态度 | 在辩论轮次中，参与者对他人 Claim 的表态。常见的产出包括同意、反对或提出具体的异议理由。 |
| **Merge Suggestion** | 合并建议 | 当发现有其他参与者提出了与自己高度相似的观点时，参与者提出的将多条 Claim 融为一体的重组建议。 |
| **Active Claims** | 活跃主张 | 经历了多轮大浪淘沙、辩论抨击后，依然存活、没有被合并或彻底推翻的观点池。 |
| **Final Vote** | 最终投票 | 辩论环节结束后的强制动作。所有参与者对 Active Claims 进行表决（支持或反对）。 |
| **Vote Threshold** | 投票阈值 | 核心配置项（如 `0.75`）。决定一条 Active Claim 能否最终作为“共识结果”被写进报告的赞成票比例门槛。 |
| **Early Stop** | 提前结束机制 | 优化机制。如果达到了最小辩论轮数（`min_rounds`），且所有人全票同意或无人再提新观点，系统会切断无意义的对话，直接触发最终投票。 |
