# Delphi 德尔菲模式

## 模式详述

`delphi`（德尔菲模式）通过机制设计强制压制“权威偏见”和“从众心理”，是一种用于形成稳定预测和技术选型的专家匿名收敛机制。

**核心逻辑：**
它完全排除了面对面的争吵或法庭对峙，而是采用“背靠背打分收敛”。所有的参与者（匿名专家）全程互相不知道对方是谁，也看不到未经加工的原始言论。他们每一轮只需要填答一份标准问卷（对陈述进行量化打分和理由简述）。一轮结束后，一位中立的 Facilitator（引导者）会收集所有问卷，剥离身份信息，合成一份单纯客观的“数据分布和分歧摘要”，再把这份匿名摘要发给所有人。专家们在看到大盘后，开始新一轮的打分修订。当所有人的打分离散度缩小至配置的收敛阈值（`convergence_threshold`）内时，即宣告达成共识。

**适用场景：**
- 评估长期技术决策或架构演进方向（如：“未来 6 个月是否应将单体拆分为事件驱动？”）。
- 多方参与的方案取舍和权重分配。
- 需要消除强话语权个体影响的文档收敛定调。

**固定宏观阶段（Phase）：**
1. `frame`: 收敛问题定义，保证所有参与者对“题目”理解一致。
2. `ingest`: 摄入相关的背景材料，形成统一的输入基准。
3. `delphi_questionnaire`: 专家们独立、匿名地填写问卷，给出初步评分与理由。
4. `delphi_summary`: 引导者汇总全局打分情况，生成抹除个人身份的聚合摘要和分歧点梳理。
5. `delphi_revision`: 专家们阅读摘要，基于大盘情况修改自己的评分与反思理由。
6. *(循环 3、4、5 步，直至达到收敛阈值或达到最大配置轮数)*。
7. `report`: 产出包含了推荐建议、收敛度评估及最终异议记录的报告。
8. `action`: 在风险管控下执行后续可能的动作。
9. `observe`: 后验观察及记录。

## 核心名词词汇表

| 英文单词 | 中文翻译 | 场景和用途 |
| :--- | :--- | :--- |
| **Participant** | 参与者 | 在此模式下充当“匿名专家”。每一轮都要根据收到的问卷独立进行打分并给出理由。 |
| **Facilitator** | 引导者 / 汇总者 | 特有角色。像一个中立的主持人，不参与观点输出或打分，只负责收集答卷并生成下一轮的匿名汇总材料。 |
| **Delphi Questionnaire** | 德尔菲问卷 | 每轮下发给专家的任务表单，要求专家针对特定的 Statement（陈述/方案）给出自己的量化评价。 |
| **Rating / Score** | 评分 / 量化打分 | 参与者必须为 Statement 给出的量化倾向（由 `rating_scale_min` 和 `max` 约束），是系统判断是否达成共识的核心底层数据指标。 |
| **Delphi Summary** | 匿名聚合摘要 | Facilitator 在每轮结束后生成的产物。包含当前的打分分布统计、主要支持理由和主要分歧点（全程抹除具体人名溯源）。 |
| **Revision** | 评分修订 | 参与者收到匿名聚合摘要后，反思是否自己忽略了某些视角（无论是从众还是坚持少数派），进而修改上一轮自己评分和理由的过程。 |
| **Convergence** | 收敛 / 共识度 | 系统自动计算的宏观指标，用来衡量所有人评分的离散程度。大家的评分越集中，收敛度数值越高。 |
| **Convergence Threshold**| 收敛阈值 | 核心配置项（如 `0.8`）。一旦系统判定当前的收敛度达到了这个基准线，就不再强行拖拽执行后续轮次，直接宣告达成共识并结束。 |
| **Dissent Summary** | 异议摘要 | 报告产出物的一部分。专门用来记录虽然大盘已经收敛（或少数服从多数），但仍有少数派坚持的、值得注意的反对理由与边缘情况。 |
| **Recommendation** | 推荐结论 | 整个 Delphi 流程达到收敛后，基于打分和摘要系统最终输出的指导性最终方案或定性结论。 |
